What is Photon PUN2?

Photon Unity Networking 2 (PUN2) is the industry-standard multiplayer networking solution for Unity. It handles the complex networking infrastructure so you can focus on gameplay:

  • Cloud Hosted: Photon provides servers worldwide — no need to manage your own infrastructure.
  • 20 Free CCU: The free tier supports 20 concurrent users — perfect for development and small indie games.
  • Cross-Platform: Works across PC, Mac, Android, iOS, WebGL, and consoles.
  • Room-Based: Players connect to rooms (lobbies) with custom properties.
  • RPCs & Synchronization: Remote Procedure Calls and automatic state sync make networking intuitive.

Setting Up Photon PUN2

Step 1: Create a Photon Account

  1. Go to photonengine.com and create a free account.
  2. Create a new application — select "Photon PUN" as the type.
  3. Copy your App ID — you'll need this in Unity.

Step 2: Install PUN2 in Unity

  1. Open Unity and go to Window → Asset Store.
  2. Search for "PUN 2 - FREE" by Exit Games.
  3. Import the package into your project.
  4. When the PUN Wizard opens, paste your App ID.

Step 3: Configure Photon Settings

Core Architecture: How Multiplayer Networking Works

Understanding the architecture is crucial before writing code. Here's how Dharmik Gohil explains the Photon networking model:

  • Client-Server Model: Photon servers relay messages between clients. There's no peer-to-peer — all data goes through Photon's cloud.
  • Master Client: One player is designated the "Master Client" — they can manage room settings and game state.
  • Rooms: Players join rooms. A room is a game session with max player limits and custom properties.
  • Lobbies: The waiting area where players browse/join rooms.
  • PhotonView: The component that marks a GameObject as networked — mandatory for any synchronized object.

Connecting to Photon — The Launcher Script

using UnityEngine;
using Photon.Pun;
using Photon.Realtime;

public class GameLauncher : MonoBehaviourPunCallbacks
{
    [SerializeField] private byte maxPlayersPerRoom = 20;
    private string gameVersion = "1.0";

    void Start()
    {
        // Connect to Photon Cloud
        PhotonNetwork.ConnectUsingSettings();
        PhotonNetwork.GameVersion = gameVersion;
        Debug.Log("Connecting to Photon...");
    }

    public override void OnConnectedToMaster()
    {
        Debug.Log("Connected to Master Server!");
        // Join the default lobby to see available rooms
        PhotonNetwork.JoinLobby();
    }

    public override void OnJoinedLobby()
    {
        Debug.Log("Joined Lobby! Ready to create or join rooms.");
    }

    public void CreateRoom(string roomName)
    {
        RoomOptions options = new RoomOptions
        {
            MaxPlayers = maxPlayersPerRoom,
            IsVisible = true,
            IsOpen = true
        };
        PhotonNetwork.CreateRoom(roomName, options);
    }

    public void JoinRoom(string roomName)
    {
        PhotonNetwork.JoinRoom(roomName);
    }

    public override void OnJoinedRoom()
    {
        Debug.Log($"Joined room: {PhotonNetwork.CurrentRoom.Name}");
        Debug.Log($"Players in room: {PhotonNetwork.CurrentRoom.PlayerCount}");
        // Load the game scene
        if (PhotonNetwork.IsMasterClient)
        {
            PhotonNetwork.LoadLevel("GameScene");
        }
    }

    public override void OnJoinRoomFailed(short returnCode, string message)
    {
        Debug.LogError($"Failed to join room: {message}");
    }
}
💡 Dharmik's Tip: Always inherit from MonoBehaviourPunCallbacks instead of MonoBehaviour for scripts that need Photon callbacks. It automatically registers and unregisters callbacks in OnEnable/OnDisable.

Player Spawning & Synchronization

This is where things get exciting. When a player joins the room, you need to spawn their character and sync it across all clients:

using UnityEngine;
using Photon.Pun;

public class PlayerSpawner : MonoBehaviour
{
    [SerializeField] private GameObject playerPrefab;
    [SerializeField] private Transform[] spawnPoints;

    void Start()
    {
        // Only spawn if we're connected to a room
        if (PhotonNetwork.IsConnectedAndReady)
        {
            SpawnPlayer();
        }
    }

    void SpawnPlayer()
    {
        // Pick a random spawn point
        int index = Random.Range(0, spawnPoints.Length);
        Vector3 spawnPos = spawnPoints[index].position;

        // PhotonNetwork.Instantiate creates the object across ALL clients
        GameObject player = PhotonNetwork.Instantiate(
            playerPrefab.name,
            spawnPos,
            Quaternion.identity
        );

        Debug.Log($"Player spawned: {PhotonNetwork.LocalPlayer.NickName}");
    }
}
⚠️ Important: The player prefab must be in a Resources folder for PhotonNetwork.Instantiate() to find it. Also, the prefab must have a PhotonView component attached.

Syncing Player Movement

using UnityEngine;
using Photon.Pun;

public class PlayerMovement : MonoBehaviourPun, IPunObservable
{
    [SerializeField] private float moveSpeed = 5f;
    private Vector3 networkPosition;
    private Quaternion networkRotation;

    void Update()
    {
        if (photonView.IsMine)
        {
            // Only the owner controls this player
            float h = Input.GetAxis("Horizontal");
            float v = Input.GetAxis("Vertical");
            Vector3 move = new Vector3(h, 0, v) * moveSpeed * Time.deltaTime;
            transform.Translate(move);
        }
        else
        {
            // Smooth interpolation for remote players
            transform.position = Vector3.Lerp(
                transform.position, networkPosition, Time.deltaTime * 10f);
            transform.rotation = Quaternion.Lerp(
                transform.rotation, networkRotation, Time.deltaTime * 10f);
        }
    }

    // This method automatically syncs data across the network
    public void OnPhotonSerializeView(PhotonStream stream, PhotonMessageInfo info)
    {
        if (stream.IsWriting)
        {
            // We own this player: send our data
            stream.SendNext(transform.position);
            stream.SendNext(transform.rotation);
        }
        else
        {
            // Remote player: receive their data
            networkPosition = (Vector3)stream.ReceiveNext();
            networkRotation = (Quaternion)stream.ReceiveNext();
        }
    }
}

Remote Procedure Calls (RPCs)

RPCs let you call functions on remote players' machines. This is how Dharmik Gohil implemented the shooting system in Mavericks Battlegrounds:

using UnityEngine;
using Photon.Pun;

public class PlayerCombat : MonoBehaviourPun
{
    [SerializeField] private int maxHealth = 100;
    private int currentHealth;

    void Start()
    {
        currentHealth = maxHealth;
    }

    public void Shoot(Player targetPlayer)
    {
        if (photonView.IsMine)
        {
            // Call TakeDamage on the target player's machine
            targetPlayer.photonView.RPC("TakeDamage", RpcTarget.All, 25);
        }
    }

    [PunRPC]
    public void TakeDamage(int damage)
    {
        currentHealth -= damage;
        Debug.Log($"Health: {currentHealth}");

        if (currentHealth <= 0)
        {
            Die();
        }
    }

    void Die()
    {
        if (photonView.IsMine)
        {
            Debug.Log("Player eliminated!");
            PhotonNetwork.Destroy(gameObject);
        }
    }
}

Room Properties & Custom Data

For a battle royale like Mavericks Battlegrounds, I needed custom room properties to track game state:

using Photon.Pun;
using Photon.Realtime;
using ExitGames.Client.Photon;

public class RoomManager : MonoBehaviourPunCallbacks
{
    public void SetGameState(string state)
    {
        Hashtable props = new Hashtable
        {
            { "GameState", state },
            { "RoundNumber", 1 },
            { "PlayersAlive", PhotonNetwork.CurrentRoom.PlayerCount }
        };
        PhotonNetwork.CurrentRoom.SetCustomProperties(props);
    }

    public override void OnRoomPropertiesUpdate(Hashtable propertiesThatChanged)
    {
        if (propertiesThatChanged.ContainsKey("GameState"))
        {
            string state = (string)propertiesThatChanged["GameState"];
            Debug.Log($"Game state changed to: {state}");
        }
    }
}

Lessons from Building Mavericks Battlegrounds — By Dharmik Gohil

Here are the key lessons I learned building a 20-player multiplayer game:

  • Minimize network traffic: Only sync what's absolutely necessary. Position and rotation — yes. Debug data — never.
  • Use PhotonTransformView: For simple object sync, the built-in component is optimized and reliable.
  • Handle disconnect gracefully: Players will disconnect. Always handle OnPlayerLeftRoom and clean up their objects.
  • Test with simulated lag: Photon's dashboard lets you simulate latency. Your game must feel smooth at 200ms ping.
  • Use object pooling: Instantiating and destroying objects over the network is expensive. Pool bullets, effects, and pickups.
  • Master Client migration: If the Master Client leaves, Photon automatically promotes another player. Handle this in OnMasterClientSwitched.
  • Room max players: For battle royale, I set max to 20. Photon's free tier supports this perfectly.
  • Use Photon's matchmaking: PhotonNetwork.JoinRandomRoom() is great for quick play. Custom room lists are better for competitive modes.
💡 Performance Tip from Dharmik: Set PhotonNetwork.SendRate = 20 and PhotonNetwork.SerializationRate = 10 for a good balance between smoothness and bandwidth. 60 sends/second is overkill for most games.

Common Pitfalls & How to Avoid Them

  1. Not checking photonView.IsMine: Always check ownership before processing input. Otherwise, every client will control every player.
  2. Scene loading issues: Use PhotonNetwork.LoadLevel() instead of SceneManager.LoadScene() for synchronized scene transitions.
  3. Prefab not in Resources: PhotonNetwork.Instantiate() only works with prefabs in a Resources folder.
  4. Not handling reconnection: Implement OnDisconnected and attempt reconnection with exponential backoff.
  5. Syncing too much data: Don't sync animations frame-by-frame. Sync trigger states and let each client play animations locally.
